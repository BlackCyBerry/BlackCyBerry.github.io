    New to Powershell?¶

    To help you get started with CyShell, you need to become somewhat comfortable with Powershell. You also need to have a functional understanding of scripting in Powershell or some other language or shell.

    Note

    If you are familiar with UNIX shells or an object oriented language, you can get proficient in Powershell very quickly. You may be able to save a lot of time by keeping some of the core concepts of Powershell in mind.

    Also, don't forget: Powershell is cross-platform, and so is CyShell - you can use it on Windows, Linux, and Mac easily.

    Powershell is pipeline-driven¶

    Just like the UNIX shells you may be familiar with, in Powershell you frequently operate on output streams coming from one command (called cmdlet in Powershell) that are passed to another cmdlet as input.

    E.g. a list of devices output by Get-CylanceDevices, piped into Powershell's own Where-Object general filtering cmdlet for filtering, whose output is then piped into Set-CylancePolicyForDevice as input objects describing the devices to assign policy for.

    Powershell is object based¶

    Instead of passing unstructured streams of strings (like you do in typical UNIX shells) you pass structured objects that have properties and methods. Powershell exposes .NET, so you have a vast library of operations at your disposal, and easy extensibility.

    In other words, you never substitute for structure by matching for e.g. character 3-12 of every line of input; instead, you'd match on $_'s (the placeholder for the current object passed by a pipeline) properties, e.g. $_.device_name.

    Powershell has guessable parameters¶

    Rather than having to know command line switches, the meaning of single-letter command line arguments, and the available values, in many places cmdlets will support static or dynamic auto-completion of arguments using the <tab> key.

    You can just guess parameters

    By typing a dash - on the command line of a cmdlet and pressing the <tab> key, you can enumerate through all available parameter names. And once you have the parameter name, you can do the same for its possible values.

    Powershell has guessable cmdlets¶

    Cmdlets generally are formed as a combination of verb and noun, with a dash in between; e.g. Connect-Cylance. There are well-known verbs (CyShell uses these: Backup, Connect, Get, Add-To, Remove-From, Block, Unblock, Enable...), followed by a noun. In case of CyShell, the noun always starts with Cylance.

    Powershell has built-in help¶

    If you want all cmdlets listed that satisfy a particular naming expression, try this one: Get-Help *-*Cylance* to get a list of all CyShell cmdlets.
