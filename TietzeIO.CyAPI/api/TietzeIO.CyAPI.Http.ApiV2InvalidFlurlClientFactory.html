Class ApiV2InvalidFlurlClientFactory

This is an intentionally invalid implementation of IFlurlClientFactory to ensure that "undefined" usage of FlurlRequests without obtaining a FlurlClient through the proper pattern first will result in an exception.

The proper pattern is to always start requests using Request(Url) to ensure that all Flurl.Http.FlurlRequest are created via ApiV2FlurlClientManager with a transaction-specific Flurl.Http.Configuration.IFlurlClientFactory
