Class ApiV2FlurlClientManager

Static class that manages multiple cached IFlurlClientFactory instances, so that ApiV2 can keep an API session-specific set of connections and logs.

The proper pattern is to always start requests via this ApiV2FlurlClientManager so that all request are created with a transaction-specific Flurl.Http.Configuration.IFlurlClientFactory for proper pooling and logging
